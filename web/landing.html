Evaluation profiles

One world. Three increasingly constrained agent interfaces.

Observation, action, knowledge, memory, seed, and inference budgets are versioned independently. Results never cross those boundaries.

Available now / privileged interface

Easy

Can the model manage a fortress when perception and UI mechanics are compressed into legal, structured controls?

Observation
Visible structured state, a bounded fort map, focused z-level maps, and recorded screen text.
Action
Audited semantic DFHack commands that create normal jobs, orders, designations, and UI-equivalent settings.
Primary capability
Long-horizon strategy, causal reasoning, spatial planning, and civilization management.
Protocol design / not yet ratified

Hard

Can the model perceive, navigate, remember, and operate the same bounded interface available to a human player?

Observation
A fixed rendered viewport. No global map, semantic coordinates, CopyScreen text, or structured inventories.
Action
Primitive keyboard and mouse inputs with fixed cadence and no semantic key catalogue.
Primary capability
Active perception, map construction, navigation, z-level reasoning, and execution.
Research horizon

Discovery

Can a fixed model learn from failed playthroughs and transfer what it discovers into unfamiliar worlds?

Observation
The Hard interface on held-out seeds and eventually held-out mechanics.
Knowledge
No guide corpus or live web; only a bounded learner-state artifact carried across declared episodes.
Primary capability
Environment learning, experimentation, recovery, and held-out transfer.
Observer Map

See the world the agent has touched, not only the frame it can see.

Spectators receive a richer, real-world atlas assembled from recorded observations and actions. It can show revealed terrain, z-levels, viewports, action footprints, verified changes, and remembered points of interest. This derived surface is never agent input unless the protocol explicitly permits it.

Measurement contract

A run is a trace, not a claim.

Every public result keeps the game, model, protocol, score, and evidence boundary inspectable.

01 / WORLD

Recorded gameplay

Per-step game frames show the actual Dwarf Fortress process at the moment of each decision.

02 / ACTION

Legal control

Governed commands are bounded, validated, provenance-tagged, and disqualify illegal or assisted progress.

03 / EFFECT

Observed change

Accepted intent is separated from jobs created, work completed, production delivered, and world state changed.

04 / SCORE

Versioned measurement

Score versions never mix, and rubric blockers remain visible beside the scalar result.

05 / COST

Inference accounting

Model calls, input tokens, output tokens, pricing snapshot, and kill reason belong with the result.

06 / REPLAY

Public failure evidence

Failures receive the same permanent replay and trace links as successful runs.
